## 1. 실험의 목적과 범위
### 1.1 프로젝트 목적
본 프로젝트의 목적은 한국사 학습 과정에서 학습자들이 자주 혼동하는 사건, 제도, 단체, 정책 등을 비교 중심으로 학습할 수 있도록 돕는 웹 프로그램을 구현하는 것이다. 기존의 한국사 학습은 개별 개념 설명 위주로 이루어지는 경우가 많아, 비슷한 개념의 공통점과 차이점을 한눈에 정리하여 이해하기 어렵다는 문제가 있었다. 본 프로젝트는 이러한 점을 보완하기 위해 두 개념을 1:1로 비교하는 구조를 중심 기능으로 설계하였다.

또한 단순히 자료를 보여주는 데 그치지 않고, 사용자가 직접 헷갈리는 개념을 입력하여 관련 비교 주제를 찾아보거나, 관련 결과가 없을 경우 학습 내용을 스스로 정리할 수 있도록 하여 사용자 주도형 학습 요소도 반영하였다.

### 1.2 프로젝트 범위
본 프로젝트는 React 프론트엔드, Flask 백엔드, MariaDB 데이터베이스를 기반으로 구현한 개인 프로젝트이며, 프론트엔드와 백엔드는 REST API 방식으로 통신하도록 구성하였다. 비교 주제 목록 조회, 비교 상세 조회, 검색, 시대 필터, 사용자 입력 기반 복습 기능을 핵심 범위로 설정하였다.
### 1.3 포함 내용
본 프로젝트에 포함된 주요 기능은 다음과 같다.

- 메인 페이지에서 프로젝트 소개 및 추천 비교 주제 제공
- 비교 주제 목록 조회
- 비교 주제 상세 조회
- 키워드 기반 검색 기능
- 시대별 필터 기능
- 사용자가 직접 개념을 입력하여 관련 비교 주제를 탐색하는 복습 페이지
- 관련 비교 주제가 없을 경우 학습 템플릿 제공
- 최근 검색어 저장
- 학습 템플릿 내용을 브라우저 localStorage에 저장 및 복원

### 1.4 불포함 내용
본 프로젝트에서 제외한 내용은 다음과 같다.

- 회원가입 및 로그인 기능
- 사용자별 서버 DB 저장 기능
- 관리자 전용 웹 화면
- 자동 채점 기능
- 퀴즈 정답 판별 기능
- 실제 배포 서버 운영

즉, 본 프로젝트는 로그인 없이 누구나 사용할 수 있는 공개형 학습 웹을 목표로 하였으며, 핵심 학습 기능을 중심으로 범위를 설정하였다.


## 2. 분석

### 2.1 주요 기능 목록
본 프로젝트의 주요 기능은 다음과 같다.

1. 메인 페이지 조회
2. 비교 주제 목록 조회
3. 비교 주제 상세 조회
4. 검색 기능
5. 시대 필터 기능
6. 복습 페이지에서 개념 입력 및 관련 비교 주제 추천
7. 비교 결과가 없을 때 학습 템플릿 제공
8. 최근 검색어 저장
9. 템플릿 입력 내용 저장 및 복원

### 2.2 유스케이스
본 프로젝트의 대표적인 유스케이스는 다음과 같다.

#### 유스케이스 1: 비교 주제 목록 조회
- 사용자는 비교 목록 페이지에 접속한다.
- 시스템은 저장된 비교 주제 목록을 조회하여 화면에 출력한다.
- 사용자는 원하는 비교 주제를 선택할 수 있다.

#### 유스케이스 2: 비교 주제 검색
- 사용자는 검색창에 키워드를 입력한다.
- 시스템은 비교 주제 제목 및 관련 개념명을 기준으로 검색한다.
- 검색 결과에 해당하는 비교 주제 목록을 사용자에게 보여준다.

#### 유스케이스 3: 시대 필터 적용
- 사용자는 특정 시대를 선택한다.
- 시스템은 선택한 시대에 해당하는 비교 주제만 필터링하여 보여준다.

#### 유스케이스 4: 비교 주제 상세 조회
- 사용자는 목록에서 특정 비교 주제를 선택한다.
- 시스템은 해당 주제의 제목, 시대, 분류, 개념 A/B, 공통점, 차이점, 시험 포인트를 상세하게 보여준다.

#### 유스케이스 5: 복습 페이지에서 개념 탐색
- 사용자는 복습 페이지에서 자신이 헷갈리는 개념을 입력한다.
- 시스템은 입력한 개념과 관련된 비교 주제를 검색하여 추천한다.

#### 유스케이스 6: 비교 결과가 없는 경우 학습 템플릿 활용
- 사용자가 입력한 개념에 대해 관련 비교 주제가 없으면 시스템은 학습 템플릿을 제공한다.
- 사용자는 시대, 관련 인물, 배경, 전개, 결과, 헷갈리는 유사 개념을 직접 작성할 수 있다.

#### 유스케이스 7: 템플릿 저장 및 복원
- 사용자가 학습 템플릿에 내용을 입력하면 시스템은 해당 내용을 localStorage에 저장한다.
- 같은 브라우저에서 동일한 개념을 다시 검색하면 이전에 저장한 템플릿 내용을 복원하여 보여준다.

### 2.3 기능 명세
주요 기능에 대한 간단한 명세는 다음과 같다.

| 기능명 | 설명 | 입력 | 처리 | 출력 |
|---|---|---|---|---|
| 비교 목록 조회 | 전체 비교 주제 목록을 조회한다 | 없음 | DB에서 비교 주제 목록 조회 | 비교 주제 리스트 |

| 검색 기능 | 키워드와 관련된 비교 주제를 검색한다 | keyword | 제목 및 개념명 LIKE 검색 | 검색 결과 리스트 |

| localStorage 저장 | 최근 검색어와 템플릿 내용을 저장한다 | 검색어, 템플릿 내용 | 브라우저 저장 | 저장 및 복원 결과 |

## 3. 설계

### 3.1 시스템 구성
본 프로젝트는 React 기반 프론트엔드, Flask 기반 백엔드, MariaDB 데이터베이스로 구성하였다. 프론트엔드는 사용자 화면 구성, 페이지 이동, 입력 처리, API 호출을 담당하고, 백엔드는 REST API를 제공하며 DB 조회 및 응답 처리를 담당한다. 데이터베이스는 개념, 비교 주제, 비교 상세, 관리자 데이터를 저장한다.

시스템 구조는 다음과 같다.

사용자 브라우저 → React 프론트엔드 → Flask REST API → MariaDB

### 3.2 페이지 구조
본 프로젝트의 주요 페이지 구성은 다음과 같다.

- `HomePage`
  - 프로젝트 소개
  - 추천 비교 주제
  - 시대별 학습 바로가기
  - 페이지 이동 버튼
- `ComparisonListPage`
  - 전체 비교 주제 목록
  - 검색창
  - 시대 필터
  - 카드형 목록 UI
- `ComparisonDetailPage`
  - 비교 주제 제목
  - 시대 및 분류
  - 개념 A/B
  - 공통점
  - 차이점
  - 시험 포인트
- `ReviewPage`
  - 개념 입력창
  - 관련 비교 주제 추천 결과
  - 결과 없을 때 학습 템플릿
  - 최근 검색어 표시
  - localStorage 기반 저장

### 3.3 데이터베이스 설계
본 프로젝트는 다음 4개의 테이블로 구성하였다.

#### 1) concept
개별 한국사 개념 정보를 저장하는 테이블이다.

- `concept_id`
- `name`
- `type`
- `era`
- `summary`
- `created_at`
- `updated_at`

#### 2) comparison_topic
두 개념의 비교 주제를 저장하는 테이블이다.

- `topic_id`
- `title`
- `concept_a_id`
- `concept_b_id`
- `era`
- `category`
- `created_at`
- `updated_at`

#### 3) comparison_detail
비교 주제의 상세 내용을 저장하는 테이블이다.

- `detail_id`
- `topic_id`
- `common_points`
- `differences`
- `exam_points`
- `created_at`
- `updated_at`

#### 4) admin
관리자 정보를 저장하는 테이블이다.

- `admin_id`
- `username`
- `password`
- `created_at`

### 3.4 테이블 관계 설명
- `comparison_topic`은 `concept`를 두 번 참조하여 개념 A와 개념 B를 연결한다.
- `comparison_detail`은 하나의 `comparison_topic`에 대한 상세 설명을 저장한다.
- `admin`은 관리자 등록, 수정, 삭제 기능 테스트를 위한 별도 테이블이다.

### 3.5 API 설계
본 프로젝트에서 구현한 주요 API는 다음과 같다.

| 메서드 | URL | 설명 |
|---|---|---|
| GET | `/api/comparisons` | 비교 주제 목록 조회 |
| GET | `/api/comparisons/<id>` | 비교 주제 상세 조회 |
| GET | `/api/comparisons/search?keyword=...` | 비교 주제 검색 |
| GET | `/api/comparisons/filter?era=...` | 시대별 필터 조회 |
| GET | `/api/concepts` | 개념 목록 조회 |
| GET | `/api/concepts/<id>` | 개념 상세 조회 |
| POST | `/api/admins` | 관리자 등록 |
| PUT | `/api/admins/<id>` | 관리자 수정 |
| DELETE | `/api/admins/<id>` | 관리자 삭제 |

### 3.6 처리 흐름 설명
검색 기능의 처리 흐름은 다음과 같다.

1. 사용자가 비교 목록 페이지의 검색창에 키워드를 입력한다.
2. 프론트엔드는 검색어를 포함한 API 요청을 백엔드로 전송한다.
3. 백엔드는 비교 주제 제목과 개념명 기준으로 DB를 조회한다.
4. 검색 결과를 JSON 형태로 반환한다.
5. 프론트엔드는 반환된 결과를 카드 목록 형태로 출력한다.

복습 페이지의 처리 흐름은 다음과 같다.

1. 사용자가 복습 페이지에서 개념을 입력한다.
2. 시스템은 관련 비교 주제를 검색한다.
3. 결과가 존재하면 추천 결과 카드를 보여준다.
4. 결과가 없으면 학습 템플릿을 출력한다.
5. 사용자가 입력한 템플릿 내용은 localStorage에 저장된다.
6. 동일 개념을 다시 검색하면 저장된 내용을 복원한다.

        │예             │아니오

         │          ┌────────────────────┐

## 4. 구현
### 4.1 개발 환경
본 프로젝트는 다음 환경에서 구현하였다.

- 운영체제: Windows
- 개발 도구: VS Code
- 데이터베이스: MariaDB
- API 테스트 도구: Thunder Client
- 버전 관리: GitHub

### 4.2 사용 기술
프로젝트 구현에 사용한 기술은 다음과 같다.

- 프론트엔드: React
- 백엔드: Flask
- 데이터베이스: MariaDB
- 언어: JavaScript, Python, SQL
- API 방식: REST API
- 기타 라이브러리: `react-router-dom`, `flask-cors`, `PyMySQL`

### 4.3 프론트엔드 구현
프론트엔드는 React를 사용하여 구현하였다. 페이지별 컴포넌트를 분리하고 `react-router-dom`을 이용해 라우팅을 구성하였다.

구현한 주요 페이지는 다음과 같다.

- `HomePage.js`
- `ComparisonListPage.js`
- `ComparisonDetailPage.js`
- `ReviewPage.js`

비교 목록 페이지에서는 API를 호출하여 목록 데이터를 가져오고, 검색 및 시대 필터 기능을 연결하였다. 비교 상세 페이지에서는 주제별 상세 정보를 출력하도록 구현하였다. 복습 페이지에서는 입력한 개념을 바탕으로 관련 비교 주제를 추천하고, 결과가 없을 경우 직접 학습 내용을 작성할 수 있도록 하였다.

### 4.4 백엔드 구현
백엔드는 Flask를 사용하여 REST API 형태로 구현하였다. Blueprint를 활용하여 라우트를 분리하였으며, 주요 파일 구성은 다음과 같다.

- `app.py`
- `db.py`
- `routes/comparison_routes.py`
- `routes/concept_routes.py`
- `routes/admin_routes.py`

DB 연결은 PyMySQL 기반으로 구성하였고, API 요청 시 MariaDB에서 데이터를 조회하여 JSON 형태로 반환하도록 구현하였다. 또한 프론트엔드와의 연동을 위해 `flask-cors`를 적용하였다.

### 4.5 데이터 저장 방식
본 프로젝트는 두 가지 저장 방식을 사용하였다.

1. **MariaDB 저장**
   - 개념 정보
   - 비교 주제 정보
   - 비교 상세 정보
   - 관리자 정보

2. **브라우저 localStorage 저장**
   - 최근 검색어
   - 복습 페이지 학습 템플릿 내용

이를 통해 서버 DB에는 공통 학습 데이터를 저장하고, 사용자가 직접 작성한 복습 내용은 로그인 없이도 브라우저 단위로 유지되도록 하였다.

## 5. 실험 및 테스트

### 5.1 테스트 환경
본 프로젝트의 테스트는 로컬 환경에서 진행하였다.

- 프론트엔드: React 개발 서버
- 백엔드: Flask 로컬 서버
- 데이터베이스: MariaDB
- API 테스트 도구: Thunder Client
- 브라우저: Chrome 또는 Edge

### 5.2 테스트 데이터
초기에는 소수의 샘플 데이터를 기반으로 기능을 확인하였고, 이후 비교 주제와 개념 데이터를 추가하여 검색 및 필터 기능의 동작을 재확인하였다. 테스트 데이터는 한국사 사건, 제도, 단체 등 비교 가능한 개념 중심으로 구성하였다.

### 5.3 기능별 테스트 결과
| 테스트 항목 | 입력/조건 | 기대 결과 | 실제 결과 |
|---|---|---|---|

| 비교 상세 조회 | 특정 `topic_id` 선택 | 상세 정보 출력 | 정상 출력 |

| 시대 필터 | `개항기` 선택 | 해당 시대의 비교 주제만 출력 | 정상 동작 |

### 5.4 테스트 결과 분석
전체적으로 핵심 기능인 목록 조회, 상세 조회, 검색, 필터, 복습 기능이 정상 동작하는 것을 확인하였다. 특히 복습 페이지는 단순 열람형 기능이 아니라, 사용자가 직접 학습 흔적을 남길 수 있는 기능으로 구현되어 프로젝트의 차별성을 높였다.

초기에는 검색 기능이 비교 주제 제목만 검색하여 결과가 제한적이었으나, 이후 관련 개념명까지 함께 검색하도록 수정하여 검색 활용성을 높였다. 또한 프론트엔드와 백엔드 연동 과정에서 CORS 문제를 확인하였고, 이를 `flask-cors` 적용으로 해결하였다.

## 6. 결론

본 프로젝트에서는 한국사 학습 과정에서 자주 혼동되는 개념들을 비교 중심으로 학습할 수 있는 웹 프로그램을 구현하였다. 단순히 자료를 나열하는 형식이 아니라, 비교 목록 조회, 상세 조회, 검색, 시대 필터 기능을 통해 사용자가 필요한 정보를 탐색할 수 있도록 하였고, 복습 페이지를 통해 사용자가 직접 학습 내용을 정리할 수 있도록 구성하였다.

특히 사용자 입력 기반 복습 페이지와 localStorage 저장 기능을 추가함으로써, 단순 자료 열람형 웹페이지를 넘어 사용자 주도형 학습 구조를 일부 반영할 수 있었다.

한계점도 존재한다. 현재는 로그인 기능이 없기 때문에 사용자별 서버 저장이 불가능하며, 브라우저 기반 localStorage 저장에 한정된다. 또한 관리자용 웹 화면, 자동 채점 기능, 배포 환경 구성 등은 구현 범위에서 제외하였다.  
추가로 프로젝트 진행 과정에서 교수님의 피드백을 반영하여 단순 자료 열람형이 아닌 사용자 활용 중심 방향으로 보완하고자 하였으나, 제한된 개발 기간과 개인 프로젝트라는 조건, 그리고 구현 난이도에 대한 부담으로 인해 아이디어를 충분히 확장하지는 못하였다. 특히 시간적 제약과 프론트엔드 구현 완성도에 대한 한계로 인해, 보다 다양한 학습 상호작용 기능이나 정교한 사용자 맞춤형 기능은 이번 프로젝트에 반영하지 못하였다.

향후 개선 방향은 다음과 같다.

- 로그인 기능 및 사용자별 학습 기록 저장
- 관리자 전용 데이터 입력/수정 화면 추가
- 퀴즈 및 자가 점검 기능 추가
- 더 다양한 한국사 비교 데이터 확장
- 실제 배포를 통한 서비스 형태 개선

종합하면, 본 프로젝트는 한국사와 컴퓨터공학을 결합한 개인 프로젝트로서, 한국사 개념 비교 학습을 지원하는 웹의 기본 구조와 핵심 기능을 구현하였다.
